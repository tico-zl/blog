Git~根据远程仓库a的tag,创建分支master,然后推到远程仓库b

文章字数 711 阅读约需 1 mins. 阅读次数 0

  • git
  • 版本控制

1、将仓库a的克隆下来

git clone https://gitlab.tico.com/test/a.git

2、更新获取tag

# 切换到a项目目录
cd a
# 更新远程tag
git fetch origin tag v1.0

3、将tag 1.0v 创建为branch分支

git branch master v1.0
# 切换创建好的分支
git ck master

4、最后将a远程地址改为b的地址

# 修改远程地址
git remote set-url origin https://newgitlab.tico.com/test/b.git
# 将代码推到远程仓库b上
git push --set-upstream origin master

5、小结
通过:git branch <new-branch-name> <tag-name> 会根据tag创建新的分支.
例如:git branch newbranch v1.0 . 会以tag v1.0创建新的分支newbranch;
